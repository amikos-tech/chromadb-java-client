# Migrating to 0.2.0

Version 0.2.0 is a major rewrite. The v1 API classes have been removed and replaced with a new builder-based v2 API.

## Breaking Changes

### Removed Classes
- `tech.amikos.chromadb.Client` - replaced by `tech.amikos.chromadb.v2.ChromaClient.builder()`
- `tech.amikos.chromadb.Collection` - replaced by `tech.amikos.chromadb.v2.Collection` interface
- `tech.amikos.chromadb.Collection.QueryResponse` - replaced by `tech.amikos.chromadb.v2.QueryResult`
- `tech.amikos.chromadb.Collection.GetResult` (inner class) - replaced by `tech.amikos.chromadb.v2.GetResult`

### Changed Packages
All v2 client types are in `tech.amikos.chromadb.v2`. Embedding functions remain in `tech.amikos.chromadb.embeddings.*` (unchanged).

### Changed Authentication
Old: `client.setDefaultHeaders()` with manual header construction
New: `ChromaClient.builder().auth(new BasicAuth(...))` with typed auth providers

## v1 to v2 Mapping

| v1 Pattern | v2 Equivalent |
|------------|---------------|
| `new Client(url)` | `ChromaClient.builder().baseUrl(url).build()` |
| `client.createCollection(name, meta, getOrCreate, ef)` | `client.getOrCreateCollection(name, CreateCollectionOptions.builder().embeddingFunction(ef).build())` |
| `collection.add(null, metadatas, documents, ids)` | `collection.add().documents(...).metadatas(...).ids(...).execute()` |
| `collection.query(texts, n, null, null, null)` | `collection.query().queryTexts(...).nResults(n).execute()` |
| `Collection.QueryResponse qr` | `QueryResult result` |
| `client.setDefaultHeaders(map)` | `ChromaClient.builder().auth(new BasicAuth(...)).build()` |
| `client.reset()` | `client.reset()` (unchanged) |
| `client.listCollections()` | `client.listCollections()` (return type changed to `List<Collection>`) |
| `client.deleteCollection(name)` | `client.deleteCollection(name)` (unchanged) |

## Before and After Examples

### Connecting

**v1 (removed):**
```java
import tech.amikos.chromadb.Client;
Client client = new Client(System.getenv("CHROMA_URL"));
```

**v2 (current):**
```java
import tech.amikos.chromadb.v2.*;
Client client = ChromaClient.builder()
        .baseUrl(System.getenv("CHROMA_URL"))
        .build();
```

### Adding Documents

**v1 (removed):**
```java
collection.add(null, metadatas, documents, ids);
```

**v2 (current):**
```java
collection.add()
        .documents("doc1", "doc2")
        .metadatas(meta1, meta2)
        .ids("id-1", "id-2")
        .execute();
```

### Querying

**v1 (removed):**
```java
Collection.QueryResponse qr = collection.query(
        Arrays.asList("search text"), 10, null, null, null);
```

**v2 (current):**
```java
QueryResult result = collection.query()
        .queryTexts("search text")
        .nResults(10)
        .include(Include.DOCUMENTS, Include.DISTANCES)
        .execute();
```

## Next Steps

See the [README](README.md) for complete v2 API usage examples including authentication, cloud connection, schema/CMEK configuration, and embedding functions.
```

All notable changes to this project will be documented in this file.

## [0.2.0] - UNRELEASED

### Added
- v2 builder-based client API (`ChromaClient.builder()`, `ChromaClient.cloud()`)
- Typed authentication providers (`BasicAuth`, `TokenAuth`, `ChromaTokenAuth`)
- Fluent record operations (`collection.add().documents(...).execute()`)
- Type-safe filter DSL (`Where`, `WhereDocument`)
- Schema and CMEK support for collection configuration
- Cloud client with Chroma Cloud authentication
- Transport customization (SSL certificates, timeouts, custom OkHttpClient)
- ID generators (UUID, ULID, SHA-256) for client-side ID generation
- Exception hierarchy (`ChromaClientException`, `ChromaServerException`, `ChromaConnectionException`)
- Default local embedding function (ONNX-based, no API key required)
- Embedding function precedence: explicit runtime > configuration > schema
- Java 8 API compatibility enforcement via animal-sniffer
- Public interface compatibility tests for API stability
- Multi-version test matrix (Chroma 1.0.0, 1.3.7, 1.5.5 x JDK 8, 11, 17)

### Changed
- Minimum supported Chroma version is now 1.0.0
- Authentication uses typed providers instead of manual header manipulation
- Collection operations use fluent builders terminated by `.execute()`
- Query results returned as `QueryResult` interface (was `Collection.QueryResponse`)
- All exceptions are unchecked (`RuntimeException`-based) for fluent API ergonomics

### Removed
- v1 API classes (`tech.amikos.chromadb.Client`, `tech.amikos.chromadb.Collection`)
- Manual auth via `setDefaultHeaders()` (use `auth()` builder method instead)

### Fixed
- DefaultEmbeddingFunction model download reliability (OkHttp with timeout and retry)
- Consistent error translation across all API endpoints
